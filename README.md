# Musubi (結び) - Programmatic Ad Platform

**Musubi** は、広告主、媒体社（パブリッシャー）、およびプラットフォーマー（管理者）を技術で結びつける、プログラマティック広告（Programmatic Advertising）のシミュレート・プラットフォームです。

---

## 🚀 クイックスタート (Developer)

### 技術スタック
- **Frontend/Backend**: Next.js 15+ (App Router)
- **Styling**: Tailwind CSS v4
- **Database**: SQLite (`better-sqlite3`) - 正規化されたリレーショナル設計
- **Charts**: Chart.js (react-chartjs-2)
- **Testing**: Vitest (Unit & Integration)

### セットアップ
```bash
# 依存関係のインストール
npm install

# サンプルデータの投入 (Onboarding)
npx tsx src/scripts/seed.ts

# テストの実行
npm test

# 開発サーバーの起動 (localhost:3000)
npm run dev
```

## 📉 システム・アーキテクチャ

Musubi は実運用の広告配信システムに基づいた論理的な階層構造を備えています。

- **Campaign (キャンペーン)**: 予算と期間（開始・終了日）を管理する最上位階層。
- **Ad Group (アドグループ)**: 入札単価（CPC）やターゲティング（デバイス、特定ドメイン）を管理。
- **Ad (広告)**: 実際の画像や遷移先URLなどのクリエイティブ素材。

データベースは中間テーブル（`ad_group_target_publishers`）を用いた多対多のリレーションで正規化されており、スケーラビリティと高速な配信クエリを実現しています。

---

## 📈 広告主 / DSP (Demand-Side Platform) 機能

- **階層別管理ダッシュボード**: キャンペーン、アドグループ、広告をそれぞれの粒度で独立して管理・編集。
- **eCPM オークション**: 入札価格 × 予測CTR に基づく動的な配信優先度決定ロジック。
- **高度な編集機能**: 既存のアドグループを別のキャンペーンへ移動させるなど、柔軟な構成変更に対応。
- **再審査トリガー**: 広告のクリエイティブ（タイトル、画像等）を変更した際の自動再審査フロー。

---

## 💰 媒体社 / SSP (Supply-Side Platform) 機能

- **簡単統合**: 専用の `<iframe>` タグを設置するだけで即座に収益化を開始。
- **収益管理**: 透明性の高い報酬分配モデル（Revenue Share）と、最低支払い金額からの申請システム。
- **データ可視化**: インプレッション、クリック、収益の実績を日別グラフで確認。

---

## 🛡️ プラットフォーム / Ad Exchange 機能

- **高速配信エンジン**: 正規化された DB 構造を活かした `EXISTS` クエリによる高度なターゲティングマッチング。
- **Click Validation (クリック検証)**: 重複クリックや不正トラフィックを検知・排除。
- **アド審査**: ネットワーク全体の品質を保つためのクリエイティブ・リンク先審査。

## 🛠 今後の展望 (Roadmap)

### 短期タスク（1年以内）
- [ ] **高度な不正検知 (IVT対策)**: ボット検知ロジックの実装とIPブラックリスト管理。
- [ ] **ads.txt 自動生成**: パブリッシャー向けの認定販売者情報の提供。

- [ ] **オーバーデリバリー対策**: 広告主の残高枯渇時のロジック改善 ([Issue #27](https://github.com/yusuke0627/adnetwork/issues/27))。

- [ ] **DMP 連携**: オーディエンスターゲティングの強化。
