# AdNetwork Prototype

本プロジェクトは、**プログラマティック広告（Programmatic Advertising）**のエコシステムをシミュレートする、フルスタック・アドネットワークのプロトタイプです。

広告主（DSP的役割）、媒体社（SSP的役割）、およびそれらを繋ぐアドエクスチェンジ機能を1つのシステムに統合し、リアルタイム取引の基礎ロジックを実装しています。

---

## 🚀 クイックスタート (Developer)

### 技術スタック
- **Runtime**: Node.js
- **Language**: TypeScript (`tsx`)
- **Framework**: Express
- **Database**: SQLite (`better-sqlite3`)
- **Template Engine**: EJS
- **Frontend**: Vanilla CSS, Chart.js

### セットアップ
```bash
npm install
npm start
```

### アーキテクチャ（役割別の分割）
- `src/services/`: 
  - `billing.ts`: クリック確定・報酬計算（レベニューシェア計算含む）
  - `stats.ts`: 統計集計（eCPM, CTR, Imps）
- `src/routes/`: 
  - `admin.ts`: **プラットフォーマー (Ad Exchange)** 機能
  - `advertiser.ts`: **広告主 (DSP - Demand-Side Platform)** 機能
  - `publisher.ts`: **媒体社 (SSP - Supply-Side Platform)** 機能
  - `delivery.ts`: 配信エンジン & トラッキング

## 📈 広告主 / DSP (Demand-Side Platform) 機能

広告主は、最適なオーディエンスに効率よく広告を入札（Bid）するための機能を備えています。

- **eCPM オークションロジック**: 単なる入札価格（Bid）だけでなく、過去の統計から予測される CTR（クリック率）を掛け合わせた「期待収益（eCPM）」に基づき、配信優先度を動的に決定します。
- **ターゲティング設定**: デバイス（PC/Mobile）および配信先ドメインのホワイトリスト指定が可能です。
- **入稿審査フロー**: 作成した広告は管理者の承認後に配信が開始されます。不承認時には理由を確認し、クリエイティブの修正が可能です。

---

## 💰 媒体社 / SSP (Supply-Side Platform) 機能

媒体社は、広告枠の収益最大化と透明性の高い報酬管理が可能です。

- **リアルタイム収益化**: 専用の `<iframe>` タグをサイトに設置するだけで、RTB（リアルタイム入札）を模したオークションにより、最も高単価な広告が自動的に配信されます。
- **収益の透明性**: 
    - **Revenue Share**: プラットフォーマーから提示された報酬率に基づき、適正な報酬が計算されます。
    - **Payment System**: 最低支払い金額（1,000円）に達すると、管理者へ支払い申請（Payout Request）が可能です。
- **実績可視化**: サイト別のインプレッション、クリック、推定収益を日別グラフで確認できます。

---

## 🛡️ プラットフォーム / Ad Exchange 機能

運営者は、取引の健全性とエコシステム全体の収益性を管理します。

- **アドクオリティ管理 (Ad Review)**: 入稿された広告の画像や遷移先を審査し、ネットワーク全体の品質を維持します。
- **マージン管理**: パブリッシャーごとに報酬率（Rev Share）を個別に設定し、プラットフォームの利益率を調整できます。
- **インバリッドトラフィック対策 (Billing)**: 重複クリックや不正なクリックを「Billing Worker」が排除し、広告主の予算とパブリッシャーの報酬を確定させます。

## 🛠 今後の展望 (Roadmap)

Gist の知見に基づき、以下の高度な機能の実装を計画しています。

- **ads.txt 対応**: 認定された販売者のみが広告枠を販売できる仕組みを導入し、取引の信頼性を向上。
- **DMP (Data Management Platform) 連携**: ユーザーの閲覧履歴に基づいた高度なオーディエンスターゲティングの実装。
- **ヘッダービディング (Header Bidding) シミュレーション**: 複数のアドネットワーク/SSPと並列で競わせ、収益を最大化するロジックの導入。
- **Vast/Vpaid 対応**: 動画広告フォーマットへの拡張。
