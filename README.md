# AdNetwork Prototype

本プロジェクトは、広告主、媒体社（パブリッシャー）、そしてプラットフォーマー（管理者）の三者を繋ぐ、運用型広告ネットワークのフルスタック・シミュレーションシステムです。

リアルタイム入札（RTB）の概念を取り入れた eCPM オークションロジック、広告審査フロー、および報酬管理機能を備えています。

---

## 🚀 クイックスタート (Developer)

開発者向けのセットアップ手順と技術仕様です。

### 技術スタック
- **Runtime**: Node.js
- **Language**: TypeScript (`tsx` による実行)
- **Framework**: Express
- **Database**: SQLite (`better-sqlite3`)
- **Template Engine**: EJS
- **Frontend**: Vanilla CSS, Chart.js (統計グラフ)

### セットアップ
```bash
# 依存関係のインストール
npm install

# サーバーの起動 (localhost:3000)
npm start

# 開発モード (nodemonによる自動再起動)
npm run dev
```

### アーキテクチャ
コードベースは「関心の分離」に基づき、以下のようにリファクタリングされています。
- `src/services/`: ビジネスロジック（報酬計算、統計集計など）
- `src/routes/`: ロール別のルーティング（Admin, Advertiser, Publisher, Delivery）
- `src/db.ts`: データベーススキーマ定義と初期データ
- `views/`: 各ロールに対応したダッシュボードテンプレート

## 📈 広告主向け機能 (Advertiser)

広告主は、予算の範囲内でターゲットユーザーに広告を届けることができます。

- **3層構造の入稿管理**: 「キャンペーン ＞ 広告グループ ＞ 広告」の構造で効率的な運用が可能です。
- **ターゲティング**: デバイス（PC/Mobile）や、特定の配信先サイト（パブリッシャー）を指定した配信が可能です。
- **eCPM オークション**: 入札価格だけでなく、広告のクリック率（CTR）を加味した「期待収益」によって配信優先度が決まる仕組みです。
- **パフォーマンス計測**: 過去7日間のインプレッション、クリック、CTRの推移をダッシュボードで視認できます。
- **ステータス管理**: 入稿した広告の審査状況（審査待ち・承認・不承認理由）を確認できます。

---

## 💰 媒体社向け機能 (Publisher)

パブリッシャーは、自身のサイトに広告枠を設置し、収益化を図ることができます。

- **広告タグの統合**: `<iframe>` タグをサイトに埋め込むだけで、自動的に最適な広告が配信されます。
- **収益管理**: 
    - **現在の残高**: 支払い申請が可能な金額。
    - **累計収益**: これまでにサイトが稼いだ総実績。
- **支払い申請システム**: 最低支払い金額（例: 1,000円）に達すると、管理者に支払いリクエストを送信できます。
- **透明性**: プラットフォーマーから設定された現在の報酬率（Revenue Share）を確認できます。

---

## 🛡️ プラットフォーマー向け機能 (Platformer / Admin)

ネットワークの運営者は、エコシステムの品質と収益性をコントロールします。

- **広告審査 (Ad Review)**: 新しく入稿された広告の画像やリンク先をチェックし、承認または不承認（理由付き）を判定します。
- **手動クリック確定処理 (Billing)**: 不正クリックを排除し、広告主の予算消費とパブリッシャーへの報酬加算を確定させます。
- **報酬率の動的設定**: パブリッシャーごとに Revenue Share（例: 70%）を個別に設定し、プラットフォームの手数料（マージン）を管理できます。
- **ネットワーク全体俯瞰**: ネットワーク全体のインプレッション、クリック、CTRの統計情報をリアルタイムで把握できます。
